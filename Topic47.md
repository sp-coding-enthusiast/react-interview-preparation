# 69. What is Hydration?

## Simple Definition

**Hydration** is the process where React **attaches JavaScript functionality to HTML that was already generated on the server.**

In simple words,

> **Hydration means React "wakes up" a static HTML page and makes it interactive.**

Before hydration, a webpage looks complete, but buttons, forms, and other interactions don't work because JavaScript hasn't connected to the HTML yet.

---

# Real-Life Analogy: A Toy Robot

Imagine buying a toy robot.

Out of the box,

it already looks like a robot.

```
Head

Body

Hands

Legs
```

But it doesn't move.

Now insert the batteries.

```
Insert Battery

↓

Robot Starts Moving
```

The robot looked complete before.

Now it becomes interactive.

Hydration works exactly like this.

---

# Why Do We Need Hydration?

Normally in a React application (Client-Side Rendering),

the browser receives almost empty HTML.

Example

```html
<body>

  <div id="root"></div>

</body>
```

Then JavaScript downloads.

```
Download JS

↓

Execute React

↓

Generate HTML

↓

Show UI
```

This takes time.

Users may initially see a blank page.

---

# Server-Side Rendering (SSR)

Instead,

the server sends ready-made HTML.

```html
<body>

<h1>Welcome</h1>

<button>

Buy Now

</button>

</body>
```

The page appears immediately.

However,

the button isn't interactive yet.

---

# Hydration Begins

After JavaScript loads,

React starts hydration.

```
Existing HTML

↓

Load JavaScript

↓

Match Components

↓

Attach Event Handlers

↓

Interactive UI
```

Now clicking the button works.

---

# What Happens During Hydration?

React:

- Reads the existing HTML
- Creates the React component tree
- Matches components with the HTML
- Attaches event listeners
- Restores React state
- Makes the application interactive

React **reuses** the existing HTML instead of recreating it.

---

# Does React Recreate the DOM?

No.

Hydration reuses the HTML generated by the server whenever it matches what React expects.

```
Server HTML

↓

React Matches It

↓

Attach Events

↓

Done
```

Creating everything again would be slower.

---

# Hydration Example in Next.js

Server

```
Generate HTML

↓

Send to Browser
```

Browser

```
Display HTML

↓

Download JS

↓

Hydrate

↓

Interactive
```

This is how frameworks like **Next.js** commonly work.

---

# Hydration Mismatch

Sometimes,

the HTML from the server doesn't match what React renders on the client.

Example

Server

```
10:00 AM
```

Browser

```
10:01 AM
```

React detects a mismatch.

This can cause warnings such as:

```
Hydration failed because the initial UI does not match what was rendered on the server.
```

---

# Common Causes

- Using `Date.now()`
- Using `Math.random()`
- Browser-only APIs like `window` during rendering
- Different data between server and client

---

# Benefits of Hydration

- Faster initial page display
- Better SEO
- Better performance
- Improved user experience
- Interactive pages after JavaScript loads

---

# Summary

- Server renders HTML.
- Browser displays it immediately.
- React attaches JavaScript behavior later.
- That process is called **Hydration**.

# 70. What is Selective Hydration?

## Simple Definition

**Selective Hydration** is a React feature that allows **only the most important parts of the page to become interactive first instead of hydrating the entire page at once.**

In simple words,

> **Instead of waking up the whole page together, React wakes up the parts the user needs first.**

This was introduced with React 18.

---

# Why Was Selective Hydration Introduced?

Imagine a webpage.

```
Header

Search Box

Navigation

Product List

Reviews

Footer
```

Without Selective Hydration,

React hydrates everything.

```
Header

↓

Search

↓

Products

↓

Reviews

↓

Footer
```

The user must wait until hydration finishes.

---

With Selective Hydration,

React can prioritize.

```
Search Box

↓

Navigation

↓

Header

↓

Products

↓

Reviews

↓

Footer
```

Now the search box works almost immediately.

---

# Real-Life Analogy: Opening a Shopping Mall

A new mall opens.

Without Selective Hydration

```

↓

```

Everyone waits.

---

With Selective Hydration

```
Open Food Court

↓

Open Pharmacy

↓

Open Supermarket

↓

Open Remaining Shops
```

Customers can start using important services immediately.

React works similarly.

React uses the **Scheduler** and **Fiber Architecture**.

# Common Interview Questions

## 1. What Is Hydration?

**Answer:**

Hydration is the process of attaching React's JavaScript logic and event handlers to HTML that was already rendered on the server, making the page interactive.

---

## 2. Why Is Hydration Needed?

**Answer:**

Server-rendered HTML is visible immediately but isn't interactive. Hydration connects React to that HTML so buttons, forms, and other interactions work.

---

## 3. Does Hydration Recreate the DOM?

**Answer:**

No.

React attempts to reuse the server-rendered HTML and attach event handlers to it. If the HTML doesn't match what React expects, it may warn about a hydration mismatch and update the affected parts.

---

## 4. What Is Selective Hydration?

**Answer:**

Selective Hydration allows React to hydrate high-priority parts of a server-rendered page before less important sections, improving responsiveness.

---

## 5. Which React Features Work with Selective Hydration?

**Answer:**

Selective Hydration works closely with:

- Fiber Architecture
- React Scheduler
- Concurrent Rendering
- Suspense
- Streaming Server Rendering

---

## 6. Why Is Selective Hydration Better?

**Answer:**

Instead of waiting for the entire page to become interactive, React prioritizes important sections first, allowing users to interact with the page sooner.

# Summary

- **Hydration** is the process of attaching React's JavaScript logic to server-rendered HTML, making it interactive.
- React reuses the existing HTML instead of rebuilding the entire DOM.
- Hydration is commonly used with **Server-Side Rendering (SSR)** frameworks like Next.js.
- **Selective Hydration**, introduced in React 18, allows React to prioritize which parts of the page become interactive first.
- It works together with **Fiber**, the **Scheduler**, **Concurrent Rendering**, and **Suspense** to improve responsiveness.
- The main benefit of Selective Hydration is that users can interact with important parts of a page before the rest of the page has finished hydrating.
